Added a scheduled production monitor for the consent-withdrawal deletion queue so
a purge that stops draining is visible to operators instead of leaving an account
in "deletion pending" indefinitely. Alerts carry job metadata only and never user
ids, health values, or secrets. The monitor reads through a new least-privilege
database accessor, so it reports the drain cron's real state and measures a stuck
worker from the same clock the queue's own reclaim uses.
